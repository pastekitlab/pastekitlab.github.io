# Doc-Forge 使用指南

## 📦 安装与配置

### 1. 加载扩展
*   在 Chrome 浏览器地址栏输入 `chrome://extensions/`。
*   开启右上角的“开发者模式”。
*   点击“加载已解压的扩展程序”，选择 `doc-forge` 项目构建后的文件夹。

### 2. 配置大模型厂商 (Provider)
首次使用时，你需要配置 AI 提供商：
1.  进入 **设置** -> **大模型厂商配置**。
2.  选择一个厂商（如阿里云 DashScope、DeepSeek 或 OpenAI）。
3.  填入你的 API Key（系统会自动加密存储）。
4.  将其设为“默认”以便快速调用。

### 3. 自定义 Prompt 与智能体
*   **Prompt 配置**：你可以修改默认的“大纲生成器”或“内容撰写者”的角色定义和任务描述。
*   **Agent 编排**：在 Agent 配置面板中，你可以创建一个新的智能体，并将多个 Prompt 串联成一个 Pipeline，实现更复杂的自动化写作流程。

## ✍️ 创作流程详解

### 第一步：生成方案大纲
1.  点击左侧侧边栏的 **“新建方案”**。
2.  在弹出的对话框中输入你的写作目标（例如：“编写一篇关于 Kubernetes 架构的深度解析文章”）。
3.  选择你配置好的智能体，点击生成。AI 将自动为你规划出 H1-H4 的多级目录。

### 第二步：编辑与填充内容
*   **左侧导航**：点击任意章节标题，编辑器会自动滚动到对应位置。双击标题可进行重命名。
*   **中间编辑区**：
    *   **预览模式**：查看渲染后的 Markdown 效果。
    *   **编辑模式**：直接修改源码。顶部会实时显示字数和 Token 消耗统计。
*   **右侧 AI 面板**：
    *   选中一段文字，点击“优化”让 AI 润色文笔。
    *   点击“扩写”让 AI 补充更多技术细节。
    *   点击“精简”去除冗余的描述。

### 第三步：管理博客元数据
如果你是 Hugo 用户，可以点击左侧侧边栏顶部的 **“博客元数据”** 卡片：
*   填写文章的简短描述 (`description`)。
*   设置分类 (`categories`) 和标签 (`tags`)，支持逗号分隔。
*   勾选“草稿状态”以控制发布。

## 🔄 Git 同步与发布

Doc-Forge 内置了强大的 Git 同步功能：
1.  进入 **设置** -> **Git 同步配置**。
2.  添加你的仓库地址（支持 GitHub, Gitee, GitLab 等）。
3.  配置分支（如 `main`）和同步目录（如 `content/posts`）。
4.  完成编辑后，点击同步按钮，文档将自动推送到你的远程仓库。

## 💡 进阶技巧

*   **Token 监控**：随时关注顶部统计栏，合理控制 API 调用成本。
*   **全文预览**：在 AI 建议面板中，点击“查看全文”可以全屏查看 AI 生成的长文本。
*   **本地隐私**：所有文档内容和配置均保存在本地 `chrome.storage` 中，确保你的技术方案不会泄露。
